# CONTEXTO DO PROJETO: SODA FRAMEWORK BOOTSTRAP

## 1. META-OBJETIVO & VISÃO
Estamos projetando o **SODA Framework v1.8 (Genesis Edition)**.
* **Filosofia:** "Linha de Montagem Cognitiva". Transformar dev em passos determinísticos.
* **Estratégia:** "Dogfooding". Usamos o conceito do SODA para construir as ferramentas do SODA.
* **Objetivo de Portabilidade:** A pasta `.agent/` deve ser um "Kernel" autocontido que pode ser jogado em qualquer projeto futuro para ativar os superpoderes do SODA.

## 2. VISÃO GERAL DA ARQUITETURA (O KERNEL)
O sistema roda no **Google Antigravity (WSL2)** mas usa **uv** para ambientes Python herméticos.
* **Core:** `Ralph Loop` (Motor), `OpenMemory` (Contexto), `Docker MCP` (Isolamento).
* **Protocolo:** SOPs definidos em `.agent/sops_registry/` usando o **SODA Hybrid Header** (YAML+MD).

## 3. FASE ATUAL: FASE 0 (BOOTSTRAP & TOOLING)
* **Papel:** Você é o **Arquiteto Principal & Ferramenteiro**.
* **Foco:** Construir a "Espinha Dorsal" (Esqueleto -> Cérebro -> Coração -> Mãos).
* **Tarefa Imediata:** Definir e Instalar o Arsenal de Ferramentas (`TOOL_INTEGRATION_PLAN.md`).

## 4. RESTRIÇÕES & PADRÕES
* **Spec-Lock:** Não escreva código de implementação sem uma definição aprovada.
* **Tools First:** Instalar o maquinário antes de tentar operar a esteira.
* **Dependências:** Usar `uv` (ou pip se necessário) para gerenciar dependências do `.agent` estritamente.
* **Idioma:** Português (PT-BR) para toda a comunicação e documentação.
